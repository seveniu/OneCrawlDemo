浏览器支持标准

WinXP Win7 OS X
IE9 A A
IE8 B B
IE7 C C
IE6 C C
Chrome16 A B A
Chrome14 A A A
Firefox10 A A A
Firefox9 A A A
Safari B B B
Opera C C C

- A级－交互和视觉完全符全设计的要求
- B级－视觉上允许有所差异，但不破坏页面的整体效果
- C级－可忽略设计上的细节，但不防碍使用

CSS 语言规范

  • 文件编码：utf-8
  • 换行符号：UNIX
  • 缩进格式：使用Tab键盘，Tab宽度为４个空格
  • 每个样式属性后（必须）加 “;”
  • 空格的使用：
    • 选择器与 { 之前（必须）要有空格
    • 属性名的 : 后（必须）要有空格
    • 属性名的 : 前（禁止）加空格
  • 使用外部样式表进行定义
  • 使用link 标签优于css中使用@import加载外部css
  • 避免使用通配符’*’
  • 不要在CSS中使用’expression’和’@import’
  • 尽可能不使用!important
  • 尽可能不使用IE滤镜

  • CSS属性定义顺序

    .content {

        /* 定位 */
        display: block;
        position: absolute;
        left: 0;
        top: 0;

        /* 盒模型 */
        width: 50px;
        height: 50px;
        margin: 10px;
        border: 1px solid black;

        /* 其他 */
        color: #efefef;
    }
    • 定位相关, 常见的有：display position left top float 等
    • 盒模型相关, 常见的有：width height margin padding border 等
    • 其他属性

  • 尽量避免使用CSS Hack, IE Hack推荐用法

    .selector {
        property: value;     /* 所有浏览器 */
        property: value\9;   /* 所有IE浏览器 */
        property: value\0;   /* IE8 */
        +property: value;    /* IE7 */
        _property: value;    /* IE6 */
        *property: value;    /* IE6-7 */
    }

CSS 命名规则

  • 使用小写、数字和’-’组合命名, 数字尽量只在表明序号时使用

    /* bad */
        .page_CANVAS{}

        /* good */
        .page-wrapper{}
        .page-canvas-0{}
        .carinfo-hover-closebtn{}

  • 使用有意义的单词进行命名，不使用短命名

    /* bad */
        .MOCS

        /* good */
        .page-wrapper
        /page-canvas-0
        .carinfo-hover-closebtn

  • 使用功能和作用进行命名

    /* bad */
        .MO-CS

        /* good */
        .content
        .visible
        .left
        .hover

  • 避免在ID或class前设置多余的元素类型

    /* bad */
        div.contemt
        p.visible
        span#test

        /* good */
        .content
        .left
        .hover

  • 使用命名空间和路径定义的class的组合进行选择，而不是遍历路径选择

    /* bad */
        .page-auto-label-car-baseinfo .note .content

        /* good */
        .page-wrapper .info-note-content // 在class定义中表明路径关系

  • 尽可能使用CSS属性的缩写形式

    /* bad */
        .content {
        margin-top: 5px;
        margin-right: 5px;
    }

        /* good */
        .content {
        margin: 5px 5px 0 0;
    }
